## Summary of changes

### New Files Created
1. **`src/Contracts/ToolDispatcherInterface.php`** - Interface defining the contract for tool dispatch:
   - `register(string $name, callable $executor)` - Register a single tool executor
   - `registerMany(array $executors)` - Register multiple tool executors
   - `has(string $name)` - Check if a tool is registered
   - `dispatch(string $name, array $arguments)` - Execute a tool with validation

2. **`src/Intent/ToolDispatcher.php`** - Implementation that:
   - Resolves tools by name from registered executors
   - Validates arguments against JSON schemas from ToolRegistry
   - Executes tools and ensures JSON-safe results
   - Wraps non-array results and handles encoding errors

3. **`tests/Unit/Intent/ToolDispatcherTest.php`** - Unit tests (9 tests, 16 assertions)

### Modified Files
1. **`src/Intent/Handlers/AbstractAgenticHandler.php`**:
   - Removed `implements ToolExecutorInterface`
   - Replaced `ToolArgumentValidator` with `ToolDispatcherInterface` 
   - Added abstract `registerToolExecutors()` method
   - `executeToolCalls()` now delegates to `$this->toolDispatcher->dispatch()`

2. **All 7 concrete handlers** updated to implement `registerToolExecutors()`:
   - `OrderStatusHandler.php`
   - `OrderRefundHandler.php`
   - `ProductStockHandler.php`
   - `EmailDraftHandler.php`
   - `AnalyticsQueryHandler.php`

## Risks / follow-ups

1. **`ToolExecutorInterface` is now orphaned**: The interface is no longer implemented by `AbstractAgenticHandler`. Consider deprecating/removing it in a future task, or updating the ADR documentation to reflect the new architecture.

2. **Backward compatibility**: Any third-party code extending `AbstractAgenticHandler` will need to implement `registerToolExecutors()` instead of `execute_tool()`. The compile-time error (abstract method not implemented) will guide migration.

3. **Optional `ToolDispatcherInterface` in constructors**: All concrete handlers accept an optional `$toolDispatcher` parameter for DI flexibility. If not provided, a new `ToolDispatcher` is created automatically.
